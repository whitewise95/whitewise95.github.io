## 개요

Java 15는 **2020년 9월**에 출시된 비LTS(단기 지원) 릴리스입니다.
Java 14에서 준비된 언어 기능을 한 단계 더 끌어올렸고, 특히 Text Blocks가 정식 기능이 되면서
문자열 템플릿 작성 생산성이 크게 개선되었습니다.

이 글은 Java 15의 핵심 변경사항을 실무 기준으로 정리합니다.

1. 출시년도
2. 주요 변경사항 요약
3. 대표 기능 설명
4. 변경사항별 코드 예시
5. 이전 버전(Java 14)과 차이점

## 1) 출시년도

| 항목 | 내용 |
|---|---|
| 버전 | Java 15 |
| 출시년도 | 2020년 |
| 릴리스 성격 | 단기 릴리스(비LTS), 문자열 기능 정식화 + JVM/보안 개선 |

## 2) 주요 변경사항 요약

| 변경사항 | 핵심 요약 | 실무 영향 |
|---|---|---|
| Text Blocks (정식) | 멀티라인 문자열을 정식 문법으로 제공 | SQL/JSON/HTML 코드 가독성 향상 |
| Sealed Classes (Preview) | 상속 허용 대상을 제한하는 타입 모델 | 도메인 모델 안정성 강화 |
| Records (2nd Preview) | 데이터 클래스 문법 보완 | DTO/응답 모델 간결화 흐름 지속 |
| Pattern Matching for `instanceof` (2nd Preview) | 타입 검사/캐스팅 패턴 보강 | 분기 코드 간결화 |
| Hidden Classes | 프레임워크용 동적 클래스 로딩 지원 강화 | 프록시/바이트코드 생성 성능/안정성 개선 |
| ZGC 개선 | Windows/macOS 지원 확대 | 저지연 GC 실험 범위 확대 |
| EdDSA 알고리즘 지원 | 현대 서명 알고리즘 표준 지원 | 보안 기능 선택지 강화 |
| Nashorn 제거 | 구 JavaScript 엔진 제거 | 레거시 스크립트 의존 점검 필요 |

## 3) 대표 기능 설명

### Text Blocks 정식화

Java 15의 대표 기능은 **Text Blocks 정식화**입니다.
이제 preview 옵션 없이 여러 줄 문자열을 선언할 수 있어,
기존의 문자열 결합(`+`)과 이스케이프 중심 코드를 크게 줄일 수 있습니다.

적용 포인트:

- SQL/JSON/HTML/GraphQL 템플릿 코드에 우선 적용
- 긴 문자열 유지보수 시 diff 품질 개선
- 팀 코드 규칙에 Text Blocks 사용 기준 반영 권장

## 4) 변경사항별 예시

### 4-1. Text Blocks (정식)

```java
public class TextBlocksJava15 {
    public static void main(String[] args) {
        String payload = """
                {
                  "event": "order.created",
                  "version": 15,
                  "enabled": true
                }
                """;

        System.out.println(payload);
    }
}
```

설명:

- Java 15부터 preview 없이 사용 가능
- JSON/SQL 문자열의 가독성과 수정 편의성이 크게 향상

### 4-2. Text Blocks + SQL 예시

```java
public class TextBlocksSqlJava15 {
    public static void main(String[] args) {
        String sql = """
                SELECT id, email, created_at
                FROM users
                WHERE status = 'ACTIVE'
                ORDER BY created_at DESC
                """;

        System.out.println(sql);
    }
}
```

설명:

- 문자열 연결 없이 쿼리 구조를 원문처럼 유지 가능
- 코드리뷰 시 쿼리 변경점을 파악하기 쉬움

### 4-3. Sealed Classes (Preview)

```java
public class SealedClassPreviewExample {
    public static void main(String[] args) {
        Payment p = new CardPayment();
        System.out.println(p.type());
    }
}

sealed interface Payment permits CardPayment, CashPayment {
    String type();
}

final class CardPayment implements Payment {
    public String type() {
        return "CARD";
    }
}

final class CashPayment implements Payment {
    public String type() {
        return "CASH";
    }
}
```

설명:

- 허용된 하위 타입을 명시해 도메인 제약을 타입 레벨에서 표현
- 무분별한 상속 확장을 줄이고 모델 안정성을 높임

### 4-4. Records / Pattern Matching (2nd Preview)

```java
public class PatternMatchingJava15 {
    public static void printData(Object obj) {
        if (obj instanceof String text) {
            System.out.println("len=" + text.length());
        } else {
            System.out.println("unknown type");
        }
    }

    public static void main(String[] args) {
        printData("java15");
        printData(123);
    }
}
```

설명:

- 타입 검사와 캐스팅을 한 줄 흐름으로 처리 가능
- 복잡한 조건 분기에서 코드 길이와 실수 가능성 감소

### 4-5. Preview 기능 컴파일/실행 (Sealed/Records/Pattern Matching)

```text
# 컴파일
javac --enable-preview --release 15 SealedClassPreviewExample.java PatternMatchingJava15.java

# 실행
java --enable-preview SealedClassPreviewExample
java --enable-preview PatternMatchingJava15
```

설명:

- Java 15 기준 sealed/records/pattern matching은 preview
- 로컬/CI/IDE 옵션을 동일하게 맞춰야 실행 불일치 방지 가능

### 4-6. Hidden Classes 사용 예시 (MethodHandles)

```java
import java.lang.invoke.MethodHandles;

public class HiddenClassExample {
    public static void main(String[] args) throws Throwable {
        MethodHandles.Lookup lookup = MethodHandles.lookup();

        // 실제 사용 시에는 바이트코드(byte[])를 전달해 hidden class를 정의
        // 여기서는 API 사용 위치를 보여주는 예시
        System.out.println("Hidden classes are for runtime-generated bytecode.");
        System.out.println(lookup.lookupClass().getName());
    }
}
```

설명:

- 프레임워크(프록시/ORM/AOP)에서 런타임 생성 클래스 처리 시 유용
- 일반 업무 코드보다 라이브러리/엔진 구현에서 효과가 큼

### 4-7. ZGC 옵션 확인 예시

```text
java -XX:+UnlockExperimentalVMOptions -XX:+UseZGC -Xms2g -Xmx2g -Xlog:gc*=info -jar app.jar
```

설명:

- Java 15에서 플랫폼 지원 범위가 넓어져 실험 환경 구성이 수월
- 적용 전엔 pause time/throughput/RSS를 함께 측정 권장

### 4-8. EdDSA 서명 예시

```java
import java.security.KeyPair;
import java.security.KeyPairGenerator;
import java.security.Signature;

public class EdDSAExample {
    public static void main(String[] args) throws Exception {
        KeyPairGenerator kpg = KeyPairGenerator.getInstance("Ed25519");
        KeyPair kp = kpg.generateKeyPair();

        Signature sig = Signature.getInstance("Ed25519");
        sig.initSign(kp.getPrivate());
        sig.update("hello-java-15".getBytes());

        byte[] signed = sig.sign();
        System.out.println("signature bytes=" + signed.length);
    }
}
```

설명:

- 현대 서명 알고리즘을 표준 JCA API로 사용 가능
- 보안 요구사항이 높은 시스템에서 선택지 확대

## 5) 이전 버전(Java 14)과 차이점

| 비교 항목 | Java 14 | Java 15 |
|---|---|---|
| Text Blocks | 2차 Preview | 정식 기능 |
| 타입 모델 | sealed class 없음 | sealed classes(Preview) 도입 |
| Record 상태 | 1차/2차 준비 단계 | 2차 Preview로 완성도 상승 |
| `instanceof` 패턴 | Preview 시작 | 2차 Preview로 보강 |
| JVM/보안 | NPE 진단, 패키징 강화 | hidden classes, EdDSA, ZGC 지원 확대 |
| 레거시 구성요소 | 일부 유지 | Nashorn 제거 |

핵심 정리:

- Java 14가 언어 기능의 "전환점"이었다면,
- Java 15는 실무에서 바로 체감되는 "정식화와 확장"에 초점을 둔 릴리스입니다.

## 실무 마이그레이션 체크리스트

| 체크 항목 | 확인 내용 |
|---|---|
| 문자열 리팩터링 | 긴 JSON/SQL/HTML 문자열을 Text Blocks로 치환 가능한지 검토 |
| preview 기능 정책 | sealed/records/pattern matching 사용 범위와 빌드 옵션 표준화 |
| 프레임워크 호환성 | 동적 프록시/바이트코드 생성 라이브러리 버전 점검 |
| 보안 알고리즘 | Ed25519/EdDSA 도입 필요성 검토 |
| JVM 옵션 점검 | Nashorn/CMS 등 제거·변경 항목 사용 여부 확인 |

## 마무리

Java 15는 언어 생산성과 런타임 유연성을 함께 끌어올린 버전입니다.
다음 글(Java 16)에서는 records 정식화, pattern matching for `instanceof` 정식화, 강한 캡슐화 강화 포인트를 중심으로 이어서 정리하겠습니다.
